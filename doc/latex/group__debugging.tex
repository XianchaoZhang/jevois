{\footnotesize\ttfamily \#define L\+D\+E\+B\+UG(\begin{DoxyParamCaption}\item[{}]{msg }\end{DoxyParamCaption})}



方便用户打印出控制台或系统日志消息的宏，\+D\+E\+B\+UG 级别 

此宏旨在与面向流的语法一起使用，用于作为参数传递给宏的所有内容。该语法一开始有点奇怪，但您很快就会习惯它。这允许在 日志中打印出任何定义了运算符 $<$$<$ 的数据类型（与 printf 样式的语法相反）。例如：


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{int} x = 3; std::string str = \textcolor{stringliteral}{"hello"}; \mbox{\hyperlink{classjevois_1_1StepRange}{jevois::StepRange<int>}} rng(0, 5, 100);}
\DoxyCodeLine{\mbox{\hyperlink{group__debugging_ga136d9d772791ddd40a7781b0f6b01dd6}{LDEBUG}}(\textcolor{stringliteral}{"x="} << x << \textcolor{stringliteral}{" and str="} << str << \textcolor{stringliteral}{" and rng="} << rng);}
\end{DoxyCode}


\begin{DoxyNote}{Note}
这是发出消息的首选方式。不要使用 printf，不要使用 cout$<$$<$\char`\"{}blah\char`\"{} 等。
\end{DoxyNote}
\begin{DoxyWarning}{Warning}
\mbox{\hyperlink{group__debugging_ga136d9d772791ddd40a7781b0f6b01dd6}{L\+D\+E\+B\+U\+G()}}、\+L\+I\+N\+F\+O() 和 \mbox{\hyperlink{group__debugging_gad364e6e39b7970e4548d99dc3c0c68cc}{L\+E\+R\+R\+O\+R()}} 中不允许使用换行符（\textquotesingle{}~\newline
\textquotesingle{} 和类似符号），因为这些可能会通过串行端口发送到 Arduino 等简单处理器，只需一个前缀（“\+D\+B\+G”、“\+I\+N\+F”、“\+E\+R\+R”）后跟消息，即可轻松解析。如果您发送多行消息，\+Je\+Vois-\/\+Inventor 可能无法运行。异常错误消息以及 \mbox{\hyperlink{group__debugging_ga07fea0c726b5acfbb6c0d5483dd15d0d}{L\+F\+A\+T\+A\+L()}} 和 \mbox{\hyperlink{Log_8H_a8e3f8ce22d435337fb988c530e098b25}{L\+T\+H\+R\+O\+W()}} 中允许使用换行符，并且多行将作为几条连续消息发送。
\end{DoxyWarning}
根据设计，如果当前日志级别低于（高于）调试级别，则不会评估您的日志消息。这意味着您不必担心浪费 C\+PU 计算不会输出的消息； 例如：


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{group__debugging_gadf127ca2262cc160830da49c37d04e85}{LINFO}}(\textcolor{stringliteral}{"CPU-\/intensive function says: "} << cpu\_intensive\_function());}
\end{DoxyCode}


如果当前日志级别为 L\+O\+G\+\_\+\+E\+R\+R，则不会运行 C\+PU 密集型函数（它仍将运行一个 \char`\"{}if\char`\"{} 语句来检查当前日志级别）。这也意味着您永远 不应假设您的日志消息将被评估。例如： \begin{DoxyVerb}@code
\end{DoxyVerb}
 int x = 42; L\+D\+E\+B\+UG(\char`\"{}x = \char`\"{} $<$$<$ (x++) ); // x 现在可能是 43 或 42，具体取决于当前日志级别... 

\begin{DoxyNote}{Note}
因为 \mbox{\hyperlink{group__debugging_ga136d9d772791ddd40a7781b0f6b01dd6}{L\+D\+E\+B\+U\+G()}} 可用于调试许多快速循环，包括通过使用 \mbox{\hyperlink{group__debugging_ga9061a9b1a920652dd863efb219c0d9d4}{J\+E\+V\+O\+I\+S\+\_\+\+T\+R\+A\+C\+E(level)}}，所以只有在构建期间定义了 J\+E\+V\+O\+I\+S\+\_\+\+L\+D\+E\+B\+U\+G\+\_\+\+E\+N\+A\+B\+LE 时才会编译它（通常，这是作为传递给 cmake 的选项完成的），否则它将被注释掉，这样就不会浪费 C\+P\+U。 
\end{DoxyNote}
