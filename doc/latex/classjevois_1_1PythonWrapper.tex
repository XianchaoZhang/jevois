辅助类，用于从 C++ 运行 Python 代码 

此类加载 Python 代码并使其可通过 \mbox{\hyperlink{classjevois_1_1PythonWrapper_a8a10f3634fb5a645d134783363a67e67}{pyinst()}} 访问。它旨在与其他类组合，这些类实际上将通过 \mbox{\hyperlink{classjevois_1_1PythonWrapper_a8a10f3634fb5a645d134783363a67e67}{pyinst()}} 执行一些 Python 代码。

这里的主要约定是，python 代码将定义一个与 python 文件名完全相同的类（不带尾随的 .py 扩展名）。该类随后可通过 \mbox{\hyperlink{classjevois_1_1PythonWrapper_a8a10f3634fb5a645d134783363a67e67}{pyinst()}} 访问。

请注意，构造函数和 \mbox{\hyperlink{classjevois_1_1PythonWrapper_a5ddf8ccc64d17a72d6eaedc450baa14b}{pythonload()}} 不会抛出错误，但如果在构造过程中发生错误，pyinst() 将会抛出错误

通常，此类由从 \mbox{\hyperlink{classjevois_1_1Component}{Component}} 继承的类继承，以创建可以运行 Python 代码的 \mbox{\hyperlink{classjevois_1_1Component}{Component}} 派生类。在这种情况下， \mbox{\hyperlink{classjevois_1_1PythonWrapper}{Python\+Wrapper}} 将在加载 Python 代码时将该同级关联注册到 \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 。在从 \mbox{\hyperlink{classjevois_1_1PythonWrapper}{Python\+Wrapper}} 继承之前从 Component（或派 生类）继承至关重要，这样当调用 \mbox{\hyperlink{classjevois_1_1PythonWrapper}{Python\+Wrapper}} 析构函数时，\+Component 基类应该尚未被销毁，并且它将从 \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 中取 消注册。
