A simple helper class for a chat box rendered in Im\+Gui. 

This is used, for example, to create interactive dialogues with large language models (L\+L\+Ms). Users can input text in an input box, and answers can be shown in the main window above. This chatbox is intended to operate fully asynchronously\+:


\begin{DoxyItemize}
\item call \mbox{\hyperlink{classjevois_1_1ChatBox_a088a47b72df7495721c292d5d5eca7a7}{get()}} to retrieve any new input messages entered by the user. This is non-\/blocking and returns immediately. The returned string will be empty except after the user presses \mbox{[}R\+E\+T\+U\+RN\mbox{]} in the input box, in which case the whole entered input will be returned.
\item call \mbox{\hyperlink{classjevois_1_1ChatBox_a91a7add4923f41476f706a2cdb04ea24}{write\+String()}} to add to the text that is written out in the main window.
\item call \mbox{\hyperlink{classjevois_1_1ChatBox_a07bfd761354cd67af7aa005c36644ec5}{draw()}} on every video frame.
\end{DoxyItemize}

Note\+: because it is primarily intended for use in python, this class is not thread-\/safe (for example, calling \mbox{\hyperlink{classjevois_1_1ChatBox_a91a7add4923f41476f706a2cdb04ea24}{write\+String()}} and \mbox{\hyperlink{classjevois_1_1ChatBox_a07bfd761354cd67af7aa005c36644ec5}{draw()}} concurrently might crash the system. In most applications, the \mbox{\hyperlink{classjevois_1_1ChatBox}{Chat\+Box}} member functions would all be called sequentially from a single thread).
