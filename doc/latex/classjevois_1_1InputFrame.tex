原始相机输入帧周围的异常安全包装器 

此包装器的操作与标准 C++11 中的 std\+:future 非常相似。用户可以通过调用 \mbox{\hyperlink{classjevois_1_1InputFrame_a2f074467becbef9303a8531199ecc60c}{get()}} 获取相机捕获的下一张图像，如果捕获尚 未完成，则可能会阻塞，如果捕获由于某种原因（例如，相机未流式传输）失败，则可能会抛出异常。图像大小和像素类型由当 前 Video\+Mapping、相机部分定义。此外，还提供了一个 \mbox{\hyperlink{classjevois_1_1InputFrame_a3ad6ba7b0e66fefb8e43944ca4e90bcf}{done()}} 函数，用户可以在完成通过 \mbox{\hyperlink{classjevois_1_1InputFrame_a2f074467becbef9303a8531199ecc60c}{get()}} 获取的图像中的像素数据后 立即使用该函数，以允许相机驱动程序再次设置底层内存缓冲区以进行捕获。如果在 \mbox{\hyperlink{classjevois_1_1InputFrame}{Input\+Frame}} 被销毁时尚未调用 \mbox{\hyperlink{classjevois_1_1InputFrame_a3ad6ba7b0e66fefb8e43944ca4e90bcf}{done()}}， 则如果已调用 \mbox{\hyperlink{classjevois_1_1InputFrame_a2f074467becbef9303a8531199ecc60c}{get()}}，它将被自动调用。在某些情况下，尽早手动调用 \mbox{\hyperlink{classjevois_1_1InputFrame_a3ad6ba7b0e66fefb8e43944ca4e90bcf}{done()}} 可能会提高帧速率，而不是让 \mbox{\hyperlink{classjevois_1_1InputFrame}{Input\+Frame}} 析构 函数执行此操作。

\mbox{\hyperlink{classjevois_1_1InputFrame}{Input\+Frame}} 实现对输入视频帧的零拷贝、零等待访问，即：


\begin{DoxyEnumerate}
\item 您通过 \mbox{\hyperlink{classjevois_1_1InputFrame_a2f074467becbef9303a8531199ecc60c}{get()}} 获取的图像的像素数据直接是内存映射的像素缓冲区，\+Je\+Vois 芯片上的硅硬件通过直接内存访问 (D\+MA) 使用 该缓冲区将像素数据从相机芯片传输到处理器内存；
\item 一旦相机硬件捕获图像，get() 就会解除阻塞并返回该图像（而不是以固定的、有规律的间隔来获取图像）。相机有多个图 像缓冲区，允许捕获一个图像缓冲区，同时将另一个图像缓冲区通过 \mbox{\hyperlink{classjevois_1_1InputFrame_a2f074467becbef9303a8531199ecc60c}{get()}} 交由处理。这些缓冲区会被回收，即一旦调用 \mbox{\hyperlink{classjevois_1_1InputFrame_a3ad6ba7b0e66fefb8e43944ca4e90bcf}{done()}}，底层缓冲区就会被发送回相机硬件以供将来捕获。 
\end{DoxyEnumerate}
