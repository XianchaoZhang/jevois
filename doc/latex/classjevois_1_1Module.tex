视觉处理模块的虚基类 

\mbox{\hyperlink{classjevois_1_1Module}{Module}} 是实现摄像头到 U\+SB 逐帧视频处理的基类。 \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 根据最终用户选择的当前 Video\+Mapping（例如，主机上的网络摄像 头查看程序设置的当前图像分辨率、格式和帧速率）实例化一个从 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 派生的类。 根据 videomappings.\+cfg 中的 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 定义和用户选择的当前 Video\+Mapping，\+Module 被加载为共享对象 (.so) 文件。

\mbox{\hyperlink{classjevois_1_1Module}{Module}} 自 Component，因此可以包含：


\begin{DoxyItemize}
\item 任意数量的 Parameter 设置
\item 任意复杂的 \mbox{\hyperlink{classjevois_1_1Component}{Component}} 对象子层次结构，用于实现各种功能。所有子组件的 Parameter 设置均可供 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 和 \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 使用。
\end{DoxyItemize}

这样，人们就可以高效地实现复杂的视觉处理管道，并大量重复使用代码。例如，人们可能首先想要实现一个 Edge\+Detector 或 Saliency 组件，并为各种阈值、特征等设置 Parameter。然后，人们可以创建任意数量的顶级对象，这些对象派生自 Module， 并且可能在子组件层次结构中包含一个或多个 Edge\+Detector、\+Saliency 等组件，模块中的实现只是将图像从一个 \mbox{\hyperlink{classjevois_1_1Component}{Component}} 路由到另一个，以创建处理管道。

从 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 派生的类应该实现最多四个函数：


\begin{DoxyItemize}
\item 当当前 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 同时指定特定的相机分辨率和格式以及 U\+SB 分辨率和格式时，\+Engine 主循环每次迭代都会调用一次 \mbox{\hyperlink{classjevois_1_1Module_aeee04cc9f45ee39774e7f2ce6116059a}{process(\+Input\+Frame \&\& inframe, Output\+Frame \&\& outframe)}}。此函数应处理接收到的输入帧并用结果填充输出帧的像素缓冲 区。在调用 \mbox{\hyperlink{classjevois_1_1Module_aeee04cc9f45ee39774e7f2ce6116059a}{process()}} 之前，已经为输入和输出图像分配了内存。由于输出图像实际上是由 U\+SB \mbox{\hyperlink{classjevois_1_1Gadget}{Gadget}} 驱动程序（最终由 Linux 内核）分配的，因此其像素内存位置无法移动（因此，不要尝试复制输出图像或用另一个图像替换它等；只需将像素数 据写入输出图像的像素阵列）。视频模式或帧速率没有限制，除非 \mbox{\hyperlink{classjevois_1_1Camera}{Camera}} 硬件支持，并且受 U\+SB 带宽限制。对于大多数实现 ，匹配输入和输出帧速率是最简单的，这意味着每次调用 \mbox{\hyperlink{classjevois_1_1Module_aeee04cc9f45ee39774e7f2ce6116059a}{process()}} 都会访问和使用提供的 \mbox{\hyperlink{classjevois_1_1InputFrame}{Input\+Frame}} 和 Output\+Frame（一 输入一输出处理管道）。但这不是强制性的。例如，用于飞行无人机的运动流计算 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 可能具有 320x240 Y\+U\+YV 53.\+0fps 输入和 100x142 G\+R\+EY 10.\+6fps 输出（由于此示例中的输出帧速率比输入低 5 倍，因此 \mbox{\hyperlink{classjevois_1_1Module_aeee04cc9f45ee39774e7f2ce6116059a}{process()}} 函数每调用 5 次才会获取 、填充和发送一次 Output\+Frame；process() 函数的实现应该跟踪这一点，例如，通过每次调用 \mbox{\hyperlink{classjevois_1_1Module_aeee04cc9f45ee39774e7f2ce6116059a}{process()}} 时递增的成员变量 ）。除了填充 \mbox{\hyperlink{classjevois_1_1OutputFrame}{Output\+Frame}} 的像素数据之外，process() 还可以使用 \mbox{\hyperlink{classjevois_1_1Module_a25fcd554bf1ee47e02c51ff1844f5482}{send\+Serial()}} 通过串行端口发送结果（例如，供连接到 Je\+Vois 平台硬件的 Arduino 使用）。
\item 当当前 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 指定特定的相机分辨率和格式，并且 U\+SB 输出格式为 N\+O\+NE 时，每个 \mbox{\hyperlink{classjevois_1_1Camera}{Camera}} 帧调用一次 \mbox{\hyperlink{classjevois_1_1Module_a81c140e6db5945679845aa9108b8ecd4}{process(\+Input\+Frame \&\& inframe)}}。此函数应处理接收到的输入帧，然后通常使用 \mbox{\hyperlink{classjevois_1_1Module_a25fcd554bf1ee47e02c51ff1844f5482}{send\+Serial()}} 将结果发送到串行端口（例 如，供连接到 Je\+Vois 平台硬件的 Arduino 使用）。对视频模式或帧速率没有限制，除非 \mbox{\hyperlink{classjevois_1_1Camera}{Camera}} 硬件支持。
\item \mbox{\hyperlink{classjevois_1_1Module_a4fa64441f85b9a23f17d972b1d9b2c51}{parse\+Serial(std\+::string const \& str, std\+::shared\+\_\+ptr$<$\+User\+Interface$>$ s)}} 允许 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 支持自定义用户命令。\+Engine 会将串行或其他它 \mbox{\hyperlink{classjevois_1_1UserInterface}{User\+Interface}} 收到的不理解的任何命令转发给此函数。您应该将其用于超出 Parameter 设置（ \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 本身已经支持）或 \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 内置命令（参见 \mbox{\hyperlink{UserCli}{命令行界面用户指南}} ）的范围。例如，可以在此处实现一个名为 \char`\"{}start\char`\"{} 的命令 ，以允许用户启动某些特定的东西。
\item supports\+Commands(std\+::ostream \& os) 应输出 \mbox{\hyperlink{classjevois_1_1Module_a4fa64441f85b9a23f17d972b1d9b2c51}{parse\+Serial()}} 支持的任何自定义命令的可读描述。当用户通过 \mbox{\hyperlink{classjevois_1_1Serial}{Serial}} 端口 输入 \char`\"{}help\char`\"{} 时，将显示这些描述。
\end{DoxyItemize}

\begin{DoxyNote}{Note}
每个模块实现文件都应包含对模块类的 \mbox{\hyperlink{Module_8H_a19a74668397740a2cd4813240e547179}{J\+E\+V\+O\+I\+S\+\_\+\+R\+E\+G\+I\+S\+T\+E\+R\+\_\+\+M\+O\+D\+U\+L\+E(\+M\+O\+D\+U\+L\+E\+N\+A\+M\+E)}} 的调用。这会创建一些纯 C 入口点，当 从动态库 (.so) 文件加载模块以实例化模块时，将使用这些入口点。有关示例，请参阅 \mbox{\hyperlink{ModuleTutorial}{关于如何为 Je\+Vois 编写新的机器视觉模块的教程}} 。 
\end{DoxyNote}
