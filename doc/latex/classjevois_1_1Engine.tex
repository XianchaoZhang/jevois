Je\+Vois processing engine -\/ 从相机传感器获取图像，对其进行处理，并通过 U\+SB 发送结果。 

\mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 协调视觉处理软件的执行。它是一个 \mbox{\hyperlink{classjevois_1_1Manager}{Manager}} ，即它是 \mbox{\hyperlink{classjevois_1_1Component}{Component}} 对象层次结构的根，它处理对其参数设置及其构造、 init()、 un\+Init() 和析构的访问。组件层次结构由根部的 \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 和用户在运行时选择的 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 组成，例如，在连接到 Je\+Vois 硬件的主机上 运行的摄像机软件上选择给定的视频格式。然后， \mbox{\hyperlink{classjevois_1_1Module}{Module}} 可以包含任意复杂的 \mbox{\hyperlink{classjevois_1_1Component}{Component}} 对象层次结构，其中包含参数设置。\+Module 派生自 Component，因此也可能有自己的参数设置。

\mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 包含以下基本元素：


\begin{DoxyItemize}
\item A \mbox{\hyperlink{classjevois_1_1VideoInput}{Video\+Input}}, 实例化为用于实时视频流的 \mbox{\hyperlink{classjevois_1_1Camera}{Camera}} 或用于处理预先录制的视频文件或图像序列的 \mbox{\hyperlink{classjevois_1_1MovieInput}{Movie\+Input}} （在算法开发期间有用， 用于测试和优化可再现的输入）；
\item A \mbox{\hyperlink{classjevois_1_1VideoOutput}{Video\+Output}}, 在 Je\+Vois 硬件平台上运行时实例化为 U\+SB \mbox{\hyperlink{classjevois_1_1Gadget}{Gadget}} 驱动程序，在有图形显示器的计算机上运行时实例化为 Video\+Display， 或者实例化为 \mbox{\hyperlink{classjevois_1_1MovieOutput}{Movie\+Output}} 将输出视频帧保存到磁盘，或者实例化为 \mbox{\hyperlink{classjevois_1_1VideoOutputNone}{Video\+Output\+None}} （如果）需要对视觉算法进行基准测试，同时 忽略与传输输出帧相关的任何工作。
\item A \mbox{\hyperlink{classjevois_1_1DynamicLoader}{Dynamic\+Loader}} 根据用户选择在运行时加载所选的视觉处理 Module；
\item 任意数量的 \mbox{\hyperlink{classjevois_1_1UserInterface}{User\+Interface}} 对象，实例化为硬件 \mbox{\hyperlink{classjevois_1_1Serial}{Serial}} 端口（平台硬件上的 4 针 J\+ST 1.\+0mm 连接器）、\+U\+SB \mbox{\hyperlink{classjevois_1_1Serial}{Serial}} 端口（在连接 Je\+Vois 硬件的主机上可见）通过 U\+S\+B），或 \mbox{\hyperlink{classjevois_1_1StdioInterface}{Stdio\+Interface}} （直接在终端启动 Je\+Vois \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 接受命令并在终端打印结果，在通用 计算机而不是平台硬件上运行时特别有用）。当在平台硬件上运行时，通常会创建两个 \mbox{\hyperlink{classjevois_1_1UserInterface}{User\+Interface}} 对象（一个硬件 \mbox{\hyperlink{classjevois_1_1Serial}{Serial}} ，一个通 过 U\+SB \mbox{\hyperlink{classjevois_1_1Serial}{Serial}} ），而在通用计算机上运行时，通常只创建一个 \mbox{\hyperlink{classjevois_1_1UserInterface}{User\+Interface}} （类型为 \mbox{\hyperlink{classjevois_1_1StdioInterface}{Stdio\+Interface}} ）在终端启动 jevois-\/daemon 并且接受命令）；
\item 从 videomappings.\+cfg 文件导入的 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 定义列表。这些定义指定了使用的 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 及其相应的 \mbox{\hyperlink{classjevois_1_1Camera}{Camera}} 设置并使用 U\+SB 输出的 模式，以及未通过 U\+SB 流式视频输出的模式（例如，当将硬件平台连接到 Arduino 时）仅有的）。
\end{DoxyItemize}

\mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 的主循环一直运行到用户决定退出为止，基本上会经历以下步骤：


\begin{DoxyItemize}
\item 创建 \mbox{\hyperlink{classjevois_1_1InputFrame}{Input\+Frame}} 对象，它是下一个可用 \mbox{\hyperlink{classjevois_1_1Camera}{Camera}} 帧的异常安全包装器。该帧可能尚未捕获。\+Input\+Frame 可以被理解为一种机制，用于在 将来当该帧可用时（即相机已捕获）来访问该帧。这与 C++11 的 std\+::future 框架非常相似。
\item 当当前 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 指定通过 U\+SB 流式传输视频帧时，还要创建一个 \mbox{\hyperlink{classjevois_1_1OutputFrame}{Output\+Frame}} 对象，该对象是下一个可用 \mbox{\hyperlink{classjevois_1_1Gadget}{Gadget}} 帧的异常安全包装器。 这也是一种用于访问 U\+SB 驱动程序提供的下一个空白视频缓冲区的机制，在通过 U\+SB 将其发送到主机之前应该填充有趣的像素数据。
\item 调用当前加载的 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 的 process() 函数，当当前 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 指定通过 U\+SB 发送某些视频输出时，可以调用 process(\+Input\+Frame, Output\+Frame) ， 或者当当前 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 指定没有视频输出时，调用 process(\+Input\+Frame)。 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 的 process() 函数抛出的任何异常都将 被捕获、报告和忽略。 process() 函数通常会通过 \mbox{\hyperlink{classjevois_1_1InputFrame}{Input\+Frame}} 包装器请求下一个可用的相机图像（此请求可能会阻塞，直到相机 传感器硬件捕获该帧），处理该图像，通过 \mbox{\hyperlink{classjevois_1_1OutputFrame}{Output\+Frame}} 包装器请求下一个可用的输出图像（当 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 指定 U\+SB 视频输出时）， 并将一些结果绘制到该输出图像中，然后将其通过 U\+SB 发送到主机，以供某些网络摄像头程序显示或由运行在该输出图像上的某些自定义 视觉软件进一步处理。此外，当前加载的 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 可以通过 \mbox{\hyperlink{classjevois_1_1UserInterface}{User\+Interface}} 端口发出消息（例如，指示找到目标的位置，以让 Arduino 知道它）。
\item 读取用户通过 \mbox{\hyperlink{classjevois_1_1UserInterface}{User\+Interface}} 端口发出的任何新命令并执行适当的命令。
\item 当用户在连接 Je\+Vois 硬件的主机上运行的网络摄像头软件中选择不同的视频模式时，处理用户更改 \mbox{\hyperlink{structjevois_1_1VideoMapping}{Video\+Mapping}} 的请求。此类请求 可能会触发当前 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 的卸载并加载新的 Module，并更改相机像素格式、图像大小等。这些更改保证在 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 的 process() 函数 未运行时发生，即 \mbox{\hyperlink{classjevois_1_1Module}{Module}} 程序员不必担心在执行 process() 函数期间图像尺寸或像素格式可能发生的变化。
\item 将通过 U\+SB 或 \mbox{\hyperlink{classjevois_1_1UserInterface}{User\+Interface}} 接收到的任何用户请求传递给实际的 \mbox{\hyperlink{classjevois_1_1Camera}{Camera}} 硬件驱动程序，以调整相机参数（例如，当用户更改网络 摄像头程序中的对比度时，该请求将通过 U\+SB 发送到 Engine，然后 \mbox{\hyperlink{classjevois_1_1Engine}{Engine}} 将其转发到 \mbox{\hyperlink{classjevois_1_1Camera}{Camera}} 硬件驱动程序）。 
\end{DoxyItemize}
