围绕要通过 U\+SB 发送的原始图像的异常安全包装器 

此包装器的操作方式与标准 C++11 中的 std\+:future 非常相似。用户可以通过调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ab0d1fea6060ee24de281934f048f07fb}{get()}} 获取下一个要通过 U\+SB 发送的内存分 配但空白的图像，如果 \mbox{\hyperlink{classjevois_1_1Gadget}{Gadget}} 仍在通过 U\+SB 发送所有缓冲区并且没有可用的空白缓冲区，则可能会阻塞，或者如果由于某种原 因（例如，\+U\+SB 断开连接，用户刚刚在其网络摄像头软件中更改了视频模式或关闭了它）获取该缓冲区失败，则可能会抛出异常 。分配的图像大小和像素类型由当前 Video\+Mapping、\+U\+SB 部分定义，即它是用户当前选择的 U\+SB 视频模式。为了节省时间，图 像缓冲区不会被清零，因此您不应假设图像填充了黑色像素，它可能包含随机像素或先前的输出帧。此外，还提供了 \mbox{\hyperlink{classjevois_1_1OutputFrame_ae6dafb6fab88b4166dc4b206c21c56a8}{send()}} 函 数，用户在将像素数据写入通过 \mbox{\hyperlink{classjevois_1_1OutputFrame_ab0d1fea6060ee24de281934f048f07fb}{get()}} 获取的图像后即可使用该函数，以允许 U\+SB 驱动程序将该图像发送到连接的主机。如果 在 \mbox{\hyperlink{classjevois_1_1OutputFrame}{Output\+Frame}} 被销毁时尚未调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ae6dafb6fab88b4166dc4b206c21c56a8}{send()}}，则如果已调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ab0d1fea6060ee24de281934f048f07fb}{get()}}，则会自动调用该函数。

\mbox{\hyperlink{classjevois_1_1OutputFrame}{Output\+Frame}} 实现了对输出视频帧的零拷贝、零等待访问，即：


\begin{DoxyEnumerate}
\item 您通过 \mbox{\hyperlink{classjevois_1_1OutputFrame_ab0d1fea6060ee24de281934f048f07fb}{get()}} 获取的图像的像素数据直接是内存映射像素缓冲区，\+Je\+Vois 芯片上的硅硬件将通过直接内存访问 (D\+MA) 使用 该缓冲区通过 U\+SB 链路将数据流出；
\item 只要您调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ae6dafb6fab88b4166dc4b206c21c56a8}{send()}}，该缓冲区就会排队等待通过 U\+SB 发送（而不是以固定的、有规律的间隔流出图像）。\+Gadget 有多个图 像缓冲区，允许一个缓冲区通过 U\+SB 流出，而另一个缓冲区则通过 \mbox{\hyperlink{classjevois_1_1OutputFrame_ab0d1fea6060ee24de281934f048f07fb}{get()}} 由您的应用程序填充。这些缓冲区是可回收的， 即一旦调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ae6dafb6fab88b4166dc4b206c21c56a8}{send()}}，底层缓冲区就会通过 U\+SB 流出，然后发送回 \mbox{\hyperlink{classjevois_1_1Gadget}{Gadget}} 以供您的代码将来访问。 
\end{DoxyEnumerate}

将 cv\+::\+Mat 转换/缩放为当前输出格式后发送的简写 

这主要针对 Python 模块编写者，因为他们可能会使用 Open\+CV 进行所有图像处理。cv\+::\+Mat 将重新缩放为与输出帧相同的尺寸。

给定的 cv\+::\+Mat 的像素格式猜测如下：
\begin{DoxyItemize}
\item 如果 img.\+type() == C\+V\+\_\+8\+U\+C3，则假设为 B\+GR 像素
\item 如果 img.\+type() == C\+V\+\_\+8\+U\+C1，则假设为 G\+R\+AY 像素
\item 如果 img.\+type() == C\+V\+\_\+8\+U\+C4，则假设为 R\+G\+BA 像素
\end{DoxyItemize}

C++ 模块编写者应坚持使用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ab0d1fea6060ee24de281934f048f07fb}{get()}}/send() 组合，因为这可提供更好的细粒度控制。请注意，cv\+::\+Mat 图像将始终被复制或转换为目标 Raw\+Image，然后通过调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ae6dafb6fab88b4166dc4b206c21c56a8}{send()}} 立即发送出去，因此用户在使用此函数后不应调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ae6dafb6fab88b4166dc4b206c21c56a8}{send()}}。此函数基本上等同于调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ab0d1fea6060ee24de281934f048f07fb}{get()}}、 将给定的 cv\+::\+Mat 转换为正确的输出格式，然后调用 \mbox{\hyperlink{classjevois_1_1OutputFrame_ae6dafb6fab88b4166dc4b206c21c56a8}{send()}}。仅当输出格式为 M\+J\+P\+EG 时才使用 quality，并且应介于 1 和 100 之间。
